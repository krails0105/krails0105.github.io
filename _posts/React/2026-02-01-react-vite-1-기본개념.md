# Introduction

---

React 프로젝트를 시작할 때 필요한 기본 개념과 프로젝트 생성 방법을 알아봅니다.

# 1. 왜 Vite인가?

---

## Create React App vs Vite

예전에는 React 프로젝트를 `create-react-app`(CRA)으로 만들었지만, 현재는 **Vite**가 표준입니다.

| 항목 | Create React App | Vite |
|------|-----------------|------|
| 개발 서버 시작 | 10-30초 | **1초 미만** |
| HMR (코드 변경 반영) | 2-5초 | **즉시** |
| 번들러 | Webpack | esbuild + Rollup |
| 현재 상태 | 유지보수 모드 (사실상 deprecated) | **활발히 개발 중** |

```
📚 CRA가 느린 이유:
- 개발 서버 시작 시 모든 파일을 번들링
- 코드 수정 시 전체 재번들링

📚 Vite가 빠른 이유:
- 브라우저의 ES 모듈을 직접 활용
- 수정된 파일만 다시 처리
```

> **참고**: React 공식 문서에서도 더 이상 CRA를 권장하지 않습니다.

# 2. Node.js와 npm

---

## Node.js란?

JavaScript를 브라우저 밖에서 실행할 수 있게 해주는 런타임 환경입니다.

```
📚 쉽게 말하면:
- JavaScript는 원래 웹 브라우저에서만 동작
- Node.js로 컴퓨터에서도 JavaScript 실행 가능
- React 개발 도구들이 Node.js 위에서 동작
```

## npm이란?

Node.js 설치 시 함께 설치되는 **패키지 매니저**입니다.

```
📚 npm의 역할:
- 다른 개발자들이 만든 코드(패키지)를 쉽게 설치
- 프로젝트의 의존성(dependencies)을 관리
- 프로젝트 스크립트(빌드, 실행 등)를 실행
```

## 설치 확인

```bash
node -v    # 예: v20.10.0
npm -v     # 예: 10.2.3
```

## Node.js 버전 관리 (nvm)

여러 프로젝트에서 다른 Node.js 버전이 필요할 때 `nvm`을 사용합니다.

```bash
# nvm 설치 후
nvm install 20       # Node.js 20 설치
nvm use 20           # Node.js 20 사용
nvm alias default 20 # 기본 버전 설정
```

# 3. 프로젝트 생성

---

## 명령어

```bash
npm create vite@latest my-react-app
```

## 명령어 분석

| 부분 | 설명 |
|------|------|
| `npm create` | 프로젝트 생성 명령어 |
| `vite@latest` | Vite 최신 버전 사용 |
| `my-react-app` | 프로젝트 폴더 이름 |

## 설정 선택

```
? Select a framework: React
? Select a variant: TypeScript + SWC   ← 권장
```

### TypeScript 선택 이유

- 코드 작성 중 오류 미리 발견
- 자동완성 기능 강력
- 현업에서 필수

### SWC란?

- Rust로 만든 초고속 컴파일러
- 기존 Babel보다 20배 이상 빠름

# 3. 의존성 설치

---

```bash
cd my-react-app
npm install
```

## npm install이 하는 일

1. `package.json`의 패키지 목록 확인
2. npm 레지스트리에서 다운로드
3. `node_modules/` 폴더에 저장
4. `package-lock.json` 생성 (버전 잠금)

## node_modules를 Git에 올리지 않는 이유

- 매우 큼 (수백 MB)
- `package.json`만 있으면 `npm install`로 재생성 가능

```bash
# .gitignore에 자동 추가됨
node_modules/
```

# 5. 개발 서버 실행

---

```bash
npm run dev
```

http://localhost:5173 에서 실행됩니다.

## 개발 서버의 장점

| 기능 | 설명 |
|------|------|
| HMR | 코드 수정 시 페이지 새로고침 없이 즉시 반영 |
| 즉시 시작 | 전체 번들링 없이 필요한 파일만 제공 |
| ES 모듈 | 브라우저 네이티브 모듈 활용 |
