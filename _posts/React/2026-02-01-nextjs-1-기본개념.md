# Introduction

---

Next.js는 React 기반 **풀스택 프레임워크**입니다. 서버 사이드 렌더링, API 라우트, 파일 기반 라우팅 등을 제공합니다.

# 1. React + Vite vs Next.js

---

| 특징 | React + Vite | Next.js |
|------|-------------|---------|
| 유형 | 클라이언트 앱 | 풀스택 프레임워크 |
| 서버 | 별도 필요 | 내장 |
| 라우팅 | 직접 설정 | 파일 기반 자동 |
| API | 별도 백엔드 필요 | 내장 |
| SEO | 추가 작업 필요 | 기본 지원 |

## 선택 가이드

```
📚 React + Vite:
- 간단한 SPA (Single Page Application)
- 백엔드가 이미 있는 경우
- 관리자 대시보드 같은 내부 툴

📚 Next.js:
- SEO가 중요한 웹사이트 (블로그, 쇼핑몰)
- 풀스택 애플리케이션
- API와 프론트엔드를 한 프로젝트에서
```

## 렌더링 방식 이해

Next.js가 제공하는 렌더링 전략들:

| 방식 | 설명 | 사용 예 |
|------|------|---------|
| **SSR** (Server-Side Rendering) | 요청마다 서버에서 HTML 생성 | 사용자별 다른 콘텐츠 |
| **SSG** (Static Site Generation) | 빌드 시 HTML 생성 | 블로그, 문서 사이트 |
| **ISR** (Incremental Static Regeneration) | 정적 페이지를 주기적으로 재생성 | 자주 바뀌는 콘텐츠 |
| **CSR** (Client-Side Rendering) | 브라우저에서 렌더링 | 인터랙티브 UI |

```
📚 기억하세요:
- SSG가 가장 빠름 (미리 만들어둔 HTML 제공)
- SSR은 항상 최신 데이터 보장
- Next.js App Router는 기본적으로 서버 컴포넌트 (자동 최적화)
```

# 2. 프로젝트 생성

---

```bash
npx create-next-app@latest my-next-app
```

## npx vs npm

| 명령어 | 동작 |
|--------|------|
| `npm install` | 패키지를 로컬에 설치 |
| `npx` | 임시 다운로드 후 실행 (설치 없이) |

## 설정 선택

```
Would you like to use TypeScript? Yes
Would you like to use ESLint? Yes
Would you like to use Tailwind CSS? Yes
Would you like your code inside a `src/` directory? Yes
Would you like to use App Router? Yes
Would you like to use Turbopack? Yes
```

| 옵션 | 설명 |
|------|------|
| TypeScript | 타입 안전성 |
| ESLint | 코드 품질 검사 |
| Tailwind | 유틸리티 CSS |
| src/ | 소스 코드 정리 |
| App Router | 최신 라우팅 방식 |
| Turbopack | 고속 번들러 |

# 3. 개발 서버 실행

---

```bash
cd my-next-app
npm run dev
```

http://localhost:3000 에서 실행됩니다.
