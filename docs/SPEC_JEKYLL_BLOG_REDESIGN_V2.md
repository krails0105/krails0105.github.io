# 기술 블로그(UI/UX) 리디자인 기획서 — V2 (Minimal Mistakes 기반 고도화)

대상: GitHub Pages + Jekyll (스크린샷 기준, **Minimal Mistakes** 테마 사용 흔적 있음)  
목표: 현재 V1 카드 UI는 “정돈”은 됐지만, **더 프로덕트스러운 완성도(타이포/여백/계층/인터랙션/다크모드/탐색 UX)**를 끌어올린다.  
제약: GitHub Pages 호환(루비 플러그인 의존 최소), 정적 CSS/JS 중심.

---

## 0) 스크린샷 기반 현재 상태(좋은 점/아쉬운 점)

### 잘 된 점
- 카드형 리스트 도입으로 스캔/가독성 개선됨
- 글 상세에 TOC, 관련글 카드, 태그 칩 등 “다음 행동” 요소가 생김
- 헤더에 Categories/Tags/검색 아이콘이 있어 네비게이션이 명확해짐

### 아직 아쉬운 점(= V2에서 해결)
- 전체가 **회색 톤**으로 평평하게 느껴짐 → “정보 위계(Visual hierarchy)”가 약함
- 카드/태그/버튼 스타일이 약간 투박하고, 간격/라인이 **디테일 부족**
- 좌측 사이드바(프로필/카테고리)가 여전히 **길고 무거움**
- 글 상세 페이지에서 **본문/TOC/메타**의 간격과 대비가 애매해 집중이 분산됨
- 하단 푸터의 큰 컬러 바(블루) 등 테마 기본 요소가 **브랜드 톤과 불일치**
- 검색 아이콘은 있으나 “검색 경험(모달/즉시 결과/키보드)”이 약할 가능성

## 1) V2 디자인 컨셉

### 컨셉 A: “Notion-like Tech Notes”
- 흰 배경(또는 아주 연한 배경) + 얇은 보더 + 큰 여백
- 본문 가독성 최우선(제목/헤딩/코드/표)
- 인터랙션은 미세하게(hover, focus, active)

### 컨셉 B: “Vercel-like Minimal”
- 강한 타이포 계층 + 얇은 구분선 + 깔끔한 버튼
- 섹션 헤더/카드 메타를 더 정돈
- 상단 네비게이션을 더 제품스럽게(검색/다크모드/단축키)

> V2에서는 A를 기본으로 하되, 헤더/버튼/태그는 B 감성으로 정리.

## 2) 화면별 개선안 (V2)

## 2.1 글로벌(공통) — “디자인 시스템” 먼저 정리
### (A) 컬러 토큰 재정의(CSS 변수)
- `--bg`, `--surface`, `--surface-2`, `--border`
- `--text`, `--muted`, `--link`
- `--accent`(1개), `--accent-weak`
- 다크모드용 세트도 같이 정의

### (B) 타이포/스페이싱 스케일
- Base font 16~18px, line-height 1.65~1.8
- 헤딩 간격(위/아래) 규칙화
- 카드/섹션 padding 규칙화: 12/16/24/32 단위

### (C) 컴포넌트 스타일 통일
- Card: radius 14~16px, border 1px, hover 시 border/translate 미세 변화
- Tag chip: 높이 24~28px, radius 999px, 텍스트 대비 개선
- Button: height 36~40px, secondary/ghost 2종
- Divider: 1px border

---

## 2.2 헤더(Top Nav) 고도화
### (A) 헤더 높이/정렬 정돈
- 로고/타이틀은 좌측, 네비는 우측. 아이콘/텍스트 baseline 정렬.
- 스크롤 시 헤더를 compact(높이 64→52) + 배경 blur(optional)

### (B) 검색 UX를 “모달”로 업그레이드 (추천)
- `/` 키를 누르면 검색 모달 오픈 (또는 `Cmd/Ctrl+K`)
- 입력 즉시 결과 8개 표시(제목+카테고리+날짜)
- `↑↓` 이동 + Enter 이동

> 구현은 Simple-Jekyll-Search + 모달 UI(JS)로 가능.

### (C) 다크모드 토글을 헤더에 추가
- 아이콘 버튼(🌙/☀️) + localStorage 저장
- prefers-color-scheme 자동 반영

---

## 2.3 홈/카테고리 리스트(글 리스트)
### (A) 카드 밀도(density) 옵션 2단계
- 기본: 현재 카드보다 약간 더 타이트(요약 2줄, 메타 한 줄)
- “Compact view” 토글(선택): 제목+메타만, 한 화면에 더 많이

### (B) 카드 정보 구조 개선
- 제목은 한 줄(또는 두 줄)로, 요약은 2줄 ellipsis
- 메타는 아이콘 최소화(과한 아이콘 제거), `Feb 09 · 6 min · Spark` 같은 포맷
- 태그는 “대표 3개만” 노출 + `+N` 배지

### (C) 카테고리/태그 필터를 상단에 고정
- 리스트 상단: 카테고리 드롭다운 + 태그 칩 + 검색 입력(또는 검색 모달로 대체)

---

## 2.4 좌측 사이드바(프로필/카테고리) 슬림화
### (A) 프로필 카드화 + 정보 축약
- 프로필 영역을 카드로 만들고, 링크는 3~4개만(Email/GitHub/RSS/About)
- 위치 등 부가 정보는 한 줄로 축약하거나 숨김(선택)

### (B) 카테고리 목록은 “Top + More”
- Top 8만 노출, 나머지는 접기(“Show all”)
- 카운트는 작은 badge로 오른쪽 정렬(현재 괜찮음)

### (C) 모바일에서는 사이드바를 Drawer로
- 햄버거 메뉴(또는 프로필 아이콘) → 사이드바 드로어 오픈
- 기본 화면은 콘텐츠에 집중

---

## 2.5 글 상세(Article) 고도화
### (A) 본문 타이포 완성도
- 제목 아래 메타 라인을 더 깔끔하게(아이콘 최소)
- 본문 폭 760~840px, 코드/표는 overflow 처리
- 인용/주의/팁(callout) 컴포넌트 제공

### (B) TOC를 더 유용하게
- 현재 위치 하이라이트(스크롤 스파이)
- 모바일에서는 상단 “On this page” 버튼 → 바텀시트/드롭다운 형태로

### (C) 독서 UX
- 상단 Reading progress bar(얇은 2px)
- 코드 블록 Copy 버튼(hover 시 노출)
- 헤딩 anchor 링크 아이콘(hover 시 표시)

### (D) 공유/푸터 정리
- Share 버튼은 더 작게(아이콘 버튼) 또는 글 하단 1줄 링크로 축약
- 사이트 푸터의 큰 컬러 바 제거/톤다운(브랜드 톤 일치)

## 3) Minimal Mistakes 기준 구현 포인트
- 테마 SCSS는 `_sass/minimal-mistakes/`에 있고,
  - 커스텀은 보통 `assets/css/main.scss`에 `@import "minimal-mistakes";` 후
  - `_sass/minimal-mistakes/custom/` 또는 `assets/css/custom.scss`로 override 가능
- 레이아웃/컴포넌트는 `_includes/`에서 override하는 방식 권장
- 플러그인 의존 없이:
  - 검색 JSON: Liquid로 생성
  - TOC/모달/다크모드/스크롤 스파이는 JS로 처리

## 5) 완료 기준(DoD)
- 라이트/다크 모두 “톤”이 일관되고 대비가 적절함
- 홈/카테고리 리스트에서 스캔이 빠르고, 카드/태그가 깔끔함
- `/` 또는 `Cmd/Ctrl+K`로 검색 모달이 열리고 즉시 결과가 뜸
- 모바일에서 사이드바가 드로어로 잘 동작하고 본문이 넓게 보임
- 글 상세에서 TOC 현재 위치 하이라이트 + reading progress bar 동작
- GitHub Pages 빌드/배포 문제 없음
