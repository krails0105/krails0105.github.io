# Claude 실행 프롬프트 — Jekyll(GitHub Pages) 블로그 UI 리디자인 V1

당신은 **Jekyll + GitHub Pages** 환경의 프론트엔드/테마 개발자입니다.  
아래 요구사항을 만족하도록 **실제 코드 변경(diff)** 기준으로 구현 계획과 파일 수정안을 제시하세요.

---

## 0) 배경/목표
- 현재 블로그는 콘텐츠는 좋지만 UI가 낡아 보입니다.
- 목표는 **가독성/탐색성/모바일 사용성**을 개선하고, 현대적인 미니멀 디자인으로 리디자인하는 것입니다.
- 제약: **GitHub Pages에서 그대로 빌드 가능**해야 합니다. (Jekyll 플러그인 의존 X 가정)

---

## 1) 구현해야 할 핵심 요구사항 (P0)

### 1.1 전체 레이아웃/타이포 개선
- 본문 max-width 확대(읽기 좋은 폭 720~820px)
- 기본 폰트/줄간격/여백 재조정
- 링크 기본 밑줄 제거 + hover 시만 강조(색 또는 underline)

### 1.2 홈/카테고리 리스트를 “카드형” UI로 변경
- 각 포스트를 card로 렌더:
  - 제목
  - 메타(날짜, 카테고리, 읽는시간 optional)
  - 1~2줄 요약
  - 태그 칩(optional)
- 카드 간 간격/테두리/hover 상태 제공
- 페이징 UI를 정돈된 버튼 형태로 개선(모바일은 단순화)

### 1.3 상단 헤더(Top Nav) 도입
- 좌측: 로고/블로그명
- 우측: About, Categories, Tags, Search, Quick Start Guide(있으면)
- 스크롤 시 헤더 높이를 줄이는 “compact” 옵션은 있으면 좋고 없어도 됨(P1)

### 1.4 검색 기능(최소 버전)
- GitHub Pages 호환 방식으로 구현:
  - 추천: **Simple-Jekyll-Search** (정적 `search.json` + JS 검색)
- 헤더의 검색 입력에서 입력하면 드롭다운으로 결과 5~10개 표시
- Enter 누르면 /search 페이지로 이동(또는 모달 유지)

### 1.5 사이드바 개선
- 폭 축소(260~300px)
- sticky 적용(데스크탑)
- 카테고리 목록이 길어지지 않게:
  - accordion 또는 “더보기”
- 중복된 Email 링크 등 정리

---

## 2) 추가 요구사항 (P1, 가능하면 함께)
- 글 상세 페이지:
  - TOC(헤딩 기반) 생성(플러그인 없이 JS로)
  - 관련 글(같은 카테고리/태그) 3~6개 카드
  - 이전/다음 글 네비게이션
- 코드 블록 스타일 개선 + copy 버튼(optional)

---

## 3) 작업 방식/제출 포맷 (중요)

### 3.1 먼저 “프로젝트 구조 탐색” 단계
- 레포에 어떤 레이아웃/인클루드/스타일 구조가 있는지 가정하지 말고,
  - `_layouts/`, `_includes/`, `assets/`를 기준으로
  - 현재 사용 중인 파일들을 추정한 뒤 “변경 대상 파일 목록”을 작성하세요.

### 3.2 그 다음 “구체적 코드 변경안”
- **변경/추가 파일 단위로** 코드 블록으로 제시하세요.
- 가능한 경우:
  - `_includes/post-card.html` 같은 컴포넌트 분리
  - CSS는 `assets/css/main.scss` (또는 기존 파일) 내에 토큰/컴포넌트 형태로 정리
  - JS는 `assets/js/*.js`로 분리

### 3.3 GitHub Pages 호환성 체크
- 사용한 기능이 GitHub Pages에서 문제 없는지 체크리스트 형태로 포함하세요.
- Ruby 플러그인 없이 동작해야 합니다.

### 3.4 DoD(완료 기준) 체크리스트
- 홈/카테고리 카드화 완료
- 헤더/검색 동작 확인
- 모바일 반응형 확인
- 접근성(포커스/대비) 기본 확인

---

## 4) 산출물
아래 3가지를 순서대로 출력하세요.

1) **구현 계획(우선순위 P0→P1)**  
2) **변경 파일 목록 + 각 파일의 역할**  
3) **코드 변경안(파일별 전체 코드 또는 핵심 diff)**

---

## 5) 참고: 디자인 가이드 요약(반영 권장)
- 카드 기반 리스트
- 과한 구분선 대신 여백/테두리/미세한 그림자
- 링크 기본 밑줄 제거 + hover 강조
- 태그는 칩(rounded) 형태
- CSS 변수로 컬러/타이포 토큰화
- (선택) 다크 모드 토글(localStorage)

이제 위 요구사항대로 작업을 시작하세요.
