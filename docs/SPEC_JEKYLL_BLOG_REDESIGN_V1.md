# 기술 블로그(UI/UX) 리디자인 기획서 (GitHub Pages + Jekyll) — V1

대상: 현재 블로그(스크린샷 기준)  
목표: “콘텐츠는 좋지만 UI가 낡아 보이는” 인상을 개선하고, **가독성/탐색성/모바일 사용성**을 크게 올린다.  
제약: **GitHub Pages에서 동작**(플러그인 최소/불가 가정), Jekyll 기본 기능 + 정적 JS/CSS만 사용.

---

## 0) 현 상태에서 보이는 문제(스크린샷 기반)

### 레이아웃/정보 구조
- 좌측 사이드바가 넓고, 본문 리스트 영역이 상대적으로 좁아 **콘텐츠가 답답**해 보임.
- **카테고리 목록이 길고**(스크롤 많음) 사용자가 “어디부터 봐야 할지” 판단이 어려움.
- 상단 헤더가 단순 텍스트 수준이라 **브랜드/네비게이션/검색 동선이 약함**.
- 리스트가 “제목+스니펫” 단조로운 패턴이라 **스캔(훑어보기) 효율이 낮음**.

### 타이포/스타일
- 제목이 **기본 링크 밑줄이 과함** → “구식/문서 느낌”.
- 전체 배경과 카드 구분이 약해 **시각적 계층(정보 위계)**이 흐림.
- 글 리스트 간 구분선/여백이 부족해 **덩어리로 뭉쳐 보임**.

### 기능/경험
- 빠른 검색/필터(태그/카테고리 내 검색) 동선이 약함.
- 최근 글/카테고리/태그/시리즈(연재) 같은 “다음 클릭” 유도 장치가 부족.
- 모바일에서 사이드바가 길면 **탐색이 더 어려워질 가능성**이 큼.

## 1) 리디자인 방향성(디자인 원칙)

1. **읽기 우선(Readability First)**  
   본문 폭, 글자 크기, 줄간격, 코드 블록/표 스타일을 최적화.
2. **탐색은 빠르게(Findability)**  
   검색 + 태그/카테고리 필터 + 시리즈/관련글로 “다음 글”을 쉽게.
3. **시각적 계층(Visual Hierarchy)**  
   제목/메타/요약/태그를 명확히 분리해서 스캔 속도 향상.
4. **현대적인 미니멀**  
   과한 장식 없이 카드/타이포/여백/색으로 세련된 인상.
5. **접근성/반응형 기본**  
   대비, 포커스, 키보드 탐색, 모바일 네비게이션을 기본 제공.

## 2) 화면별 개선안

## 2.1 홈(Recent posts) / 카테고리 리스트(Posts list)

### (A) 리스트를 “카드형”으로 개편
- 각 포스트 아이템을 `card`로 구성:
  - 제목(밑줄 제거, hover 시 underline 또는 색 변화)
  - 메타: 날짜 · 카테고리 · 읽는 시간(옵션) · 태그(칩)
  - 1~2줄 요약(ellipsis)
- 카드 간 간격/경계(미세한 shadow 또는 border)로 “덩어리감” 제거

**기대 효과**: 스캔 효율↑, 세련된 인상↑, 클릭률↑

### (B) “정렬/필터” 컨트롤 추가(상단)
- 정렬: 최신순(기본) / 오래된순 / (옵션) 인기순(조회수 있으면)
- 필터: 카테고리 드롭다운, 태그 칩, 키워드 검색 입력

> GitHub Pages 제약을 고려해, “정렬/필터”는  
> 1) 정적 페이지에서 가능한 수준은 Liquid로 제공하거나  
> 2) JS로 클라이언트 사이드 필터링(추천)으로 구현.

### (C) 페이징 UI 개선
- 현재 페이지 강조는 유지하되 버튼을 더 작고 정돈된 스타일로
- 모바일에서는 “Prev/Next”만 보이고, 숫자 페이지는 접기(가독성)

---

## 2.2 좌측 사이드바(프로필/카테고리)

### (A) 사이드바 폭 축소 + Sticky
- 데스크탑: 좌측 260~300px, 본문 max-width 확대
- 스크롤 시 사이드바는 sticky(프로필/카테고리)로 유지

### (B) 카테고리 “접기/펼치기(accordion)”
- 기본은 상위 몇 개(예: Top 8)만 노출 + “더보기”
- 또는 카테고리 그룹(예: Backend / Data / Infra / Frontend)로 묶기
- 카테고리 내 포스트 수(count)는 유지하되 글자 크기/대비 개선

### (C) 프로필 영역 정리
- 중복된 Email 버튼/링크 정리
- GitHub, Email, RSS, About 정도만 남기고 아이콘+라벨 정렬
- 상태 문구(“Never fall behind”)는 작은 서브텍스트로

---

## 2.3 상단 헤더(글로벌 네비게이션)

### (A) Top Nav 도입
- 좌측: 로고/블로그명
- 우측: About · Categories · Tags · Search · (옵션) Quick Start Guide
- 스크롤 시 compact 헤더(높이 줄이기)

### (B) Search 기본 제공(추천)
- GitHub Pages 환경에서 쉬운 선택:
  - **Simple-Jekyll-Search**(정적 JSON 인덱스 + JS 검색)
  - 또는 lunr.js 기반 검색(구현 난이도↑)

**검색 결과 UI**: 모달/드롭다운으로 즉시 결과 노출, Enter로 검색 페이지 이동.

---

## 2.4 글 상세(Article page)

### (A) 읽기 경험
- 본문 폭: 720~820px 내에서 최적화
- 폰트: 시스템 폰트 + 코드 폰트 분리
- 문단 간격/헤딩 간격 재조정, 리스트/인용/테이블 기본 스타일 개선

### (B) TOC(목차) 제공
- 페이지 오른쪽(데스크탑) 또는 상단(모바일) TOC
- JS로 헤딩(h2/h3) 파싱해 생성(플러그인 없이 가능)

### (C) “다음 행동” 유도
- 관련 글(같은 카테고리/태그) 3~6개 카드
- 이전/다음 글 네비게이션
- 글 상단: 태그 칩 클릭 가능

### (D) 코드 블록/Callout 개선
- 코드: padding, line-height, 가로 스크롤, copy 버튼(옵션)
- Callout: Tip/Note/Warning 박스(간단한 CSS 클래스)

---

## 3) 디자인 시스템(토큰) 제안

### 컬러(예시)
- 배경: `--bg`, 카드: `--surface`, 테두리: `--border`
- 텍스트: `--text`, 서브: `--muted`
- 강조색(Accent): `--accent` (1개만 정해 일관성 유지)

### 타이포
- Base: 16~18px, line-height 1.6~1.8
- 제목: 굵기/크기 단계화 (h1/h2/h3)
- 링크: 기본 밑줄 제거, hover에서만 underline

### 컴포넌트
- Card, TagChip, Button, Input, Pagination, Divider, Badge

### 다크 모드(추천)
- CSS 변수 기반 테마
- 토글 버튼 + localStorage 저장

## 5) Jekyll/GitHub Pages 구현 가이드(파일 단위)

> 테마에 따라 파일명은 다를 수 있음. 아래는 일반적인 구조 가이드.

### 수정/추가될 가능성이 큰 파일
- `_layouts/default.html` : 전체 레이아웃/헤더/푸터 포함
- `_layouts/home.html` 또는 `index.html` : 홈 리스트 UI
- `_layouts/category.html`(또는 custom) : 카테고리 리스트 UI
- `_layouts/post.html` : 글 상세 UI(TOC/related)
- `_includes/`  
  - `header.html`, `sidebar.html`, `post-card.html`, `search.html`, `toc.html`
- `assets/css/main.scss` : CSS 변수/컴포넌트 스타일
- `assets/js/`  
  - `search.js`(Simple-Jekyll-Search), `toc.js`, `theme.js`(다크모드)
- `search.json` : 검색 인덱스(정적 생성)
- `_config.yml` : 사이트 메타/컬렉션/퍼머링크 등

### 검색(Simple-Jekyll-Search) 구성(예시)
- `search.json`에 포스트 데이터를 Liquid로 노출
- `assets/js/search.js`에서 인덱스 fetch 후 필터링

## 6) 완료 기준(Definition of Done)

- 홈/카테고리 리스트에서:
  - 카드 UI + 메타/태그 표시 + hover 상태
  - 모바일에서도 보기 좋고, 사이드바가 방해되지 않음
- 상단 헤더:
  - 글로벌 네비게이션 + 검색 동작
- 글 상세:
  - 본문 타이포/코드/표 가독성 개선
  - TOC 표시 + 관련 글/이전다음
- 접근성:
  - 포커스 스타일, 대비, 키보드 탐색 기본 동작
- GitHub Pages에서 빌드/배포 문제 없음(플러그인 의존 X)
